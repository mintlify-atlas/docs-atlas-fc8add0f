The Marble editor is a powerful, distraction-free writing environment built on Tiptap. It combines rich text editing with markdown support, AI assistance, and real-time collaboration features.

## Editor interface

The editor is designed for focused content creation:

- **Title field** - Large, prominent text area for your post title
- **Content area** - Spacious editor canvas with prose styling
- **Floating toolbar** - Context-aware formatting options
- **Slash commands** - Quick access to all content blocks
- **Sidebar** - Post settings, metadata, and publishing options

<Info>
  The editor automatically saves your work as you type. Look for the "Saved" indicator in the header.
</Info>

## Text formatting

Format text using keyboard shortcuts or the floating toolbar that appears when you select text:

### Basic formatting

| Format | Shortcut | Description |
|--------|----------|-------------|
| **Bold** | `Cmd/Ctrl + B` | Make text bold |
| *Italic* | `Cmd/Ctrl + I` | Italicize text |
| <u>Underline</u> | `Cmd/Ctrl + U` | Underline text |
| ~~Strikethrough~~ | `Cmd/Ctrl + Shift + X` | Strike through text |
| `Code` | `Cmd/Ctrl + E` | Inline code formatting |

### Advanced formatting

- **Text color** - Change text color from the toolbar
- **Highlight** - Highlight text with customizable colors
- **Superscript** - For mathematical notation (x²)
- **Subscript** - For chemical formulas (H₂O)
- **Links** - Add hyperlinks with `Cmd/Ctrl + K`

<Note>
  Multiple formats can be combined. For example, you can have bold, italic, and colored text simultaneously.
</Note>

## Headings

Organize content with hierarchical headings:

- **Heading 1** - `Cmd/Ctrl + Alt + 1` - Main sections
- **Heading 2** - `Cmd/Ctrl + Alt + 2` - Subsections
- **Heading 3** - `Cmd/Ctrl + Alt + 3` - Sub-subsections

Or type `#` for H1, `##` for H2, `###` for H3 at the start of a line.

<Info>
  Headings automatically generate anchor links for table of contents and deep linking.
</Info>

## Lists

Create organized, hierarchical content:

### Bullet lists

Type `-` or `*` followed by space, or use `Cmd/Ctrl + Shift + 8`:

- First item
- Second item
  - Nested item (press Tab to indent)
  - Another nested item
- Back to top level (press Shift + Tab to outdent)

### Numbered lists

Type `1.` followed by space, or use `Cmd/Ctrl + Shift + 7`:

1. First step
2. Second step
   1. Sub-step (press Tab)
   2. Another sub-step
3. Third step

### Task lists

Create interactive checklists with `/task`:

- [ ] Incomplete task
- [x] Completed task
- [ ] Another task
  - [ ] Nested task

Click checkboxes to toggle completion.

    Add your code. Syntax highlighting is applied automatically.

Supported languages include:
- JavaScript, TypeScript, JSX, TSX

- Bash, Shell, SQL

<Info>
  Code blocks use [lowlight](https://github.com/wooorm/lowlight) for syntax highlighting with the same engine as GitHub.
</Info>

## Images

Add images to your posts in multiple ways:

### Upload new image

<Steps>
  <Step title="Trigger image insert">
    Type `/image`, click the image button, or drag and drop an image file.
  </Step>
  <Step title="Choose upload">
    Select **Upload image** from the menu.
  </Step>
  <Step title="Select file">
    Choose an image file from your computer (JPG, PNG, WebP, GIF).
  </Step>
  <Step title="Wait for upload">
    The image uploads to Cloudflare R2 and is inserted into your post.
  </Step>
  <Step title="Add caption (optional)">
    Click below the image to add a caption.
  </Step>
</Steps>

### Insert from library

<Steps>
  <Step title="Open image menu">
    Type `/image` or click the image button.
  </Step>
  <Step title="Browse library">
    Click **Browse library** to see previously uploaded images.
  </Step>
  <Step title="Select image">
    Click an image to insert it into your post.
  </Step>
</Steps>

### Image features

- **Captions** - Add descriptive captions below images
- **Alt text** - Set accessibility descriptions
- **Resizing** - Drag corners to resize images
- **Alignment** - Left, center, or right alignment

<Info>
  Images are automatically stored in your media library and can be reused in other posts.
</Info>

## Markdown support

The editor supports markdown input and export:

### Markdown shortcuts

Type markdown syntax and it's automatically formatted:

- `**bold**` → **bold**
- `*italic*` → *italic*
- `~~strike~~` → ~~strikethrough~~
- `[link](url)` → hyperlink
- `` `code` `` → inline code
- `> quote` → blockquote

### Paste markdown

Paste markdown content directly into the editor:

1. Copy markdown text from anywhere
2. Paste into the editor (`Cmd/Ctrl + V`)
3. Markdown is automatically converted to formatted content

### Export as markdown

Export your post as markdown:

1. Open the post in the editor
2. Click the menu icon (⋯) in the header
3. Select **Export as markdown**
4. A `.md` file is downloaded

<Info>
  Markdown export preserves all formatting, including headings, lists, links, and code blocks.
</Info>

## AI assistance

Marble includes optional AI-powered features to improve your writing:

### Enabling AI

<Steps>
  <Step title="Navigate to editor settings">
    Go to **Settings** > **Editor** in your workspace.
  </Step>
  <Step title="Enable AI">
    Toggle **AI assistance** on.
  </Step>
  <Step title="Save settings">
    AI features are now available in the editor.
  </Step>
</Steps>

<Warning>
  AI features may require additional configuration and incur costs. Contact your workspace owner for details.
</Warning>

### AI features

When enabled, AI can help with:

- **Writing suggestions** - Improve clarity and style
- **Grammar checking** - Catch errors and typos
- **Readability analysis** - Optimize for your audience
- **Content expansion** - Generate ideas and outlines

<Note>
  AI features are workspace-level settings. All team members share the same configuration.
</Note>

## Editor preferences

Customize the editor experience:

### Workspace settings

Go to **Settings** > **Editor** to configure:

- **AI assistance** - Enable or disable AI features
- **Auto-save interval** - How often content is saved
- **Default status** - Draft or published for new posts

### User preferences

Personal editor preferences (saved per user):

- **Font size** - Adjust editor text size
- **Line height** - Change spacing for readability
- **Focus mode** - Hide sidebars for distraction-free writing

<Note>
  User preferences are saved in your browser and apply across all workspaces.
</Note>

## Tips for great content

<Accordion title="Use headings for structure">
  Break content into sections with H2 and H3 headings. This improves readability and SEO.
</Accordion>

<Accordion title="Keep paragraphs short">
  Aim for 2-3 sentences per paragraph. Short paragraphs are easier to read on screens.
</Accordion>

<Accordion title="Add visual breaks">
  Use images, videos, code blocks, and lists to break up long text sections.
</Accordion>

<Accordion title="Write clear link text">
  Instead of "click here," use descriptive link text: "read our API documentation."
</Accordion>

<Accordion title="Optimize images">
  Compress and resize images before uploading to improve page load times.
</Accordion>

<Accordion title="Use code blocks for commands">
  Format terminal commands, code snippets, and configuration in code blocks with syntax highlighting.
</Accordion>

<Accordion title="Add alt text to images">
  Always provide descriptive alt text for accessibility and SEO.
</Accordion>
