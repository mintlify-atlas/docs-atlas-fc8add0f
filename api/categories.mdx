## List categories

<CodeGroup>
```bash GET /v1/categories
curl -X GET "https://api.marblecms.com/v1/categories?limit=10&page=1" \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</CodeGroup>

Retrieve a paginated list of all categories.

### Query parameters

<ParamField query="limit" type="integer" default="10">
  Number of categories per page (1-100)
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number
</ParamField>

### Response

<ResponseField name="categories" type="array">
  Array of category objects
  <Expandable title="Category object">
    <ResponseField name="id" type="string">
      Unique category identifier
    </ResponseField>
    <ResponseField name="name" type="string">
      Category name
    </ResponseField>
    <ResponseField name="slug" type="string">
      URL-friendly category slug
    </ResponseField>
    <ResponseField name="description" type="string" nullable>
      Category description
    </ResponseField>
    <ResponseField name="count" type="object">
      Post count
      <Expandable title="Count object">
        <ResponseField name="posts" type="integer">
          Number of published posts in this category
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  Pagination metadata
  <Expandable title="Pagination object">
    <ResponseField name="limit" type="integer">
      Items per page
    </ResponseField>
    <ResponseField name="currentPage" type="integer">
      Current page number
    </ResponseField>
    <ResponseField name="nextPage" type="integer" nullable>
      Next page number or null
    </ResponseField>
    <ResponseField name="previousPage" type="integer" nullable>
      Previous page number or null
    </ResponseField>
    <ResponseField name="totalPages" type="integer">
      Total number of pages
    </ResponseField>
    <ResponseField name="totalItems" type="integer">
      Total number of items
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>
```json 200
{
  "categories": [
    {
      "id": "cryitfjp2345kl05weoybfk9",
      "name": "Technology",
      "slug": "technology",
      "description": "Tech news and tutorials",
      "count": {
        "posts": 15
      }
    },
    {
      "id": "cryitfjp2345kl05weoyabc1",
      "name": "Design",
      "slug": "design",
      "description": "Design tips and inspiration",
      "count": {
        "posts": 8
      }
    }
  ],
  "pagination": {
    "limit": 10,
    "currentPage": 1,
    "nextPage": null,
    "previousPage": null,
    "totalPages": 1,
    "totalItems": 2
  }
}
```
</ResponseExample>

## Get category by slug or ID

<CodeGroup>
```bash GET /v1/categories/{identifier}
curl -X GET "https://api.marblecms.com/v1/categories/technology" \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</CodeGroup>

Retrieve a single category by its ID or slug.

### Path parameters

<ParamField path="identifier" type="string" required>
  Category ID or slug
</ParamField>

### Response

<ResponseField name="category" type="object">
  Category object with all fields
  <Expandable title="Category object">
    <ResponseField name="id" type="string">
      Category ID
    </ResponseField>
    <ResponseField name="name" type="string">
      Category name
    </ResponseField>
    <ResponseField name="slug" type="string">
      Category slug
    </ResponseField>
    <ResponseField name="description" type="string" nullable>
      Category description
    </ResponseField>
    <ResponseField name="count" type="object">
      Post count
      <Expandable title="Count object">
        <ResponseField name="posts" type="integer">
          Number of published posts
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>
```json 200
{
  "category": {
    "id": "cryitfjp2345kl05weoybfk9",
    "name": "Technology",
    "slug": "technology",
    "description": "Tech news and tutorials",
    "count": {
      "posts": 15
    }
  }
}
```

```json 404
{
  "error": "Category not found",
  "message": "The requested category does not exist"
}
```
</ResponseExample>

## Create category

<CodeGroup>
```bash POST /v1/categories
curl -X POST "https://api.marblecms.com/v1/categories" \
  -H "Authorization: Bearer YOUR_PRIVATE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Technology",
    "slug": "technology",
    "description": "Tech news and tutorials"
  }'
```
</CodeGroup>

Create a new category. Requires a private API key (starts with `msk_`).

### Request body

<ParamField body="name" type="string" required>
  Category name
</ParamField>

<ParamField body="slug" type="string" required>
  URL-friendly slug (must be unique within workspace)
</ParamField>

<ParamField body="description" type="string">
  Category description
</ParamField>

### Response

<ResponseField name="category" type="object">
  Created category
  <Expandable title="Category object">
    <ResponseField name="id" type="string">
      Category ID
    </ResponseField>
    <ResponseField name="name" type="string">
      Category name
    </ResponseField>
    <ResponseField name="slug" type="string">
      Category slug
    </ResponseField>
    <ResponseField name="description" type="string" nullable>
      Category description
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>
```json 201
{
  "category": {
    "id": "cryitfjp2345kl05weoybfk9",
    "name": "Technology",
    "slug": "technology",
    "description": "Tech news and tutorials"
  }
}
```

```json 409
{
  "error": "Slug already in use",
  "message": "A category with this slug already exists in this workspace"
}
```

```json 403
{
  "error": "Forbidden",
  "message": "Write operations require a private API key (msk_...). Public keys are read-only."
}
```
</ResponseExample>

## Update category

<CodeGroup>
```bash PATCH /v1/categories/{identifier}
curl -X PATCH "https://api.marblecms.com/v1/categories/technology" \
  -H "Authorization: Bearer YOUR_PRIVATE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Engineering",
    "description": "Engineering articles and tutorials"
  }'
```
</CodeGroup>

Update an existing category by ID or slug. All fields are optional. Requires a private API key.

### Path parameters

<ParamField path="identifier" type="string" required>
  Category ID or slug
</ParamField>

### Request body

All fields are optional. Only provided fields will be updated.

<ParamField body="name" type="string">
  Category name
</ParamField>

<ParamField body="slug" type="string">
  URL-friendly slug (must be unique within workspace)
</ParamField>

<ParamField body="description" type="string">
  Category description
</ParamField>

### Response

<ResponseField name="category" type="object">
  Updated category
</ResponseField>

<ResponseExample>
```json 200
{
  "category": {
    "id": "cryitfjp2345kl05weoybfk9",
    "name": "Engineering",
    "slug": "engineering",
    "description": "Engineering articles and tutorials"
  }
}
```

```json 404
{
  "error": "Category not found",
  "message": "The requested category does not exist"
}
```
</ResponseExample>

## Delete category

<CodeGroup>
```bash DELETE /v1/categories/{identifier}
curl -X DELETE "https://api.marblecms.com/v1/categories/technology" \
  -H "Authorization: Bearer YOUR_PRIVATE_API_KEY"
```
</CodeGroup>

Delete a category by ID or slug. Cannot delete a category that has posts assigned to it. Requires a private API key.

### Path parameters

<ParamField path="identifier" type="string" required>
  Category ID or slug
</ParamField>

### Response

<ResponseField name="id" type="string">
  ID of the deleted category
</ResponseField>

<ResponseExample>
```json 200
{
  "id": "cryitfjp2345kl05weoybfk9"
}
```

```json 400
{
  "error": "Category has posts",
  "message": "This category has 15 post(s) assigned to it. Reassign or delete them before deleting this category."
}
```

```json 404
{
  "error": "Category not found",
  "message": "The requested category does not exist"
}
```
</ResponseExample>
