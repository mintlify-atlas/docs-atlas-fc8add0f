# Quickstart

This guide will get you from zero to fetching your first post via the Marble API in just a few minutes.

## Sign up and create a workspace

<Steps>
  <Step title="Create your account">
    Visit [marblecms.com](https://marblecms.com) and sign up with Google or GitHub OAuth.
  </Step>
  
  <Step title="Create a workspace">
    After signing in, you'll be prompted to create your first workspace. A workspace is a container for all your content - think of it as a separate site or project.
    
    Give your workspace a name like "My Blog" or "Product Updates".
  </Step>
  
  <Step title="Create your first category">
    Categories help organize your content. Create a category like "Announcements" or "Blog Posts".
    
    Navigate to **Settings** → **Categories** → **New Category** and enter:
    - **Name**: `Announcements`
    - **Slug**: `announcements` (auto-generated)
    - **Description**: Optional description
  </Step>
</Steps>

## Create your first post

<Steps>
  <Step title="Navigate to posts">
    Click **Posts** in the sidebar, then click **New Post**.
  </Step>
  
  <Step title="Write your content">
    Fill in the post details:
    
    - **Title**: `Welcome to Marble`
    - **Slug**: `welcome-to-marble` (auto-generated from title)
    - **Category**: Select the category you created
    - **Content**: Use the rich text editor to write your post
    
    The Tiptap editor supports:
    - Rich text formatting (bold, italic, lists)
    - Code blocks with syntax highlighting
    - Images and media embeds
    - Tables and blockquotes
  </Step>
  
  <Step title="Publish the post">
    Click **Publish** to make your post live. You can also save as a draft if you're not ready to publish.
    
    <Note>
      Only published posts are returned by the API by default. Draft posts can be fetched using the `status=draft` query parameter.
    </Note>
  </Step>
</Steps>

## Get your API key

<Steps>
  <Step title="Navigate to API keys">
    Go to **Settings** → **API Keys** in your workspace.
  </Step>
  
  <Step title="Create an API key">
    Click **Create API Key** and configure:
    
    - **Name**: `Production` or `Development`
    - **Type**: 
      - **Public** - Read-only access (safe for client-side code)
      - **Private** - Full read/write access (keep secret)
    
    <Warning>
      Private API keys can create, update, and delete content. Never expose them in client-side code or commit them to version control.
    </Warning>
  </Step>
  
  <Step title="Copy your API key">
    Copy the API key and store it securely. You won't be able to see it again.
    
    ```bash
    # Add to your .env file
    MARBLE_API_KEY=your_api_key_here
    ```
  </Step>
</Steps>

## Fetch your content via API

Now let's fetch the post you just created using the REST API.

<CodeGroup>
```bash cURL
curl https://api.marblecms.com/v1/posts \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch('https://api.marblecms.com/v1/posts', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const { posts, pagination } = await response.json();

console.log('Total posts:', pagination.totalItems);
console.log('First post:', posts[0].title);
```

```python Python
import requests

response = requests.get(
    'https://api.marblecms.com/v1/posts',
    headers={'Authorization': 'Bearer YOUR_API_KEY'}
)

data = response.json()
print(f"Total posts: {data['pagination']['totalItems']}")
print(f"First post: {data['posts'][0]['title']}")
```

```typescript TypeScript
interface Post {
  id: string;
  slug: string;
  title: string;
  content: string;
  description: string;
  publishedAt: string;
  category: {
    id: string;
    name: string;
    slug: string;
  };
}

interface PostsResponse {
  posts: Post[];
  pagination: {
    limit: number;
    currentPage: number;
    totalPages: number;
    totalItems: number;
  };
}

const response = await fetch('https://api.marblecms.com/v1/posts', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const data: PostsResponse = await response.json();
```
</CodeGroup>

## Fetch a single post

You can fetch a specific post by its slug or ID:

<CodeGroup>
```bash cURL
curl https://api.marblecms.com/v1/posts/welcome-to-marble \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch(
  'https://api.marblecms.com/v1/posts/welcome-to-marble',
  {
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY'
    }
  }
);

const { post } = await response.json();
console.log(post.title);
```
</CodeGroup>

## Filter and paginate

The API supports powerful filtering and pagination:

```bash
# Get posts from a specific category
curl "https://api.marblecms.com/v1/posts?categories=announcements" \
  -H "Authorization: Bearer YOUR_API_KEY"

# Get page 2 with 20 posts per page
curl "https://api.marblecms.com/v1/posts?page=2&limit=20" \
  -H "Authorization: Bearer YOUR_API_KEY"

# Search posts
curl "https://api.marblecms.com/v1/posts?query=welcome" \
  -H "Authorization: Bearer YOUR_API_KEY"

# Get featured posts only
curl "https://api.marblecms.com/v1/posts?featured=true" \
  -H "Authorization: Bearer YOUR_API_KEY"

# Get posts with specific tags
curl "https://api.marblecms.com/v1/posts?tags=javascript,react" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Content formats

By default, content is returned as HTML. You can request Markdown format:

```bash
curl "https://api.marblecms.com/v1/posts?format=markdown" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Rate limits

<Note>
  API keys have rate limits to ensure fair usage:
  
  - **Public keys**: 100 requests per minute
  - **Private keys**: 1000 requests per minute
  
  Rate limit headers are included in every response:
  - `X-RateLimit-Limit` - Your rate limit
  - `X-RateLimit-Remaining` - Requests remaining
  - `X-RateLimit-Reset` - Time when the limit resets
</Note>
